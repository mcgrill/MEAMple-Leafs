robockey player code


Our robots run using the M2 microcontroller developed by Jonathan Fiene, a professor at the University of Pennsylvania. The m2 runs C code, wth custom header files for USB, wifi, mWii communication, and much more.

Robot M2 Code
*The meampleleafs code was programmed using AVR studio 5. AVR 5 generates a very specific compiling structure, so use care if you're trying to use a custom Make file. You can learn more about compiling on Mac here.



debugging code


It's often tedious trying to figure out where things went wrong. To help expedite the process, we've put together a couple of test scripts.

Debugging Code

  • 'photoTransistorLivePlotting' reads in all of the sensor data from the four phototransistors placed around the robot. In Matlab, the ADC values are plotted in realtime so you can figure out if you're getting accurate data and determine if your sensors are placed correctly.
  • 'wiiBlobPlottingLogging' reads in the raw wii sensor data and plots these on the graph as dots in Matlab. It also has capability to run these raw values through a location-determining script (in our case, the_NAMEEN_deal) and plot the position and orientation on a graphed rink. You can also save raw IR data logs from here to run through different scripts later.
  • 'testWiiBlobLogs' loads raw wii blob data amd runs it through a localization script. The location is then plotted on a graphed rink.
